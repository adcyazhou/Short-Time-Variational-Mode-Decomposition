# 单孔漏斗爆破多通道 STVMD Notebook 设计

## 目标

新建一个自包含的 Jupyter Notebook，读取仓库根目录下的 `5m.TXT`、`10m.TXT` 和 `15m.TXT`，分别对 Tran、Vert、Long 三个方向执行动态多通道短时变分模态分解（STVMD）。每次分解把 5m、10m、15m 作为三个同步通道，并输出模态、动态中心频率、时频谱、重构诊断，以及频谱区域与瞬时频率轨迹的对应图。

实现以仓库中的 `main_STVMD.ipynb` 为主，并以论文 *Short-Time Variational Mode Decomposition*（Signal Processing 238 (2026) 110203）核对算法定义。2022 年 STMVMD 论文仅用于理解用户提供的参考图，不采用其中的参数尺度。

## 输入数据与时间处理

三个 TXT 文件均包含 Tran、Vert、Long 三列振动速度数据，单位为 `mm/s`，采样率为 4096 Hz，预触发长度为 0.500 s。

三份记录的有效样本数分别为：

- 5m：33493 点
- 10m：14336 点
- 15m：23176 点

多通道 STVMD 要求各通道长度一致。主分析截取到共同有效长度 14336 点，不做前部或尾部补零。统一相对时间轴为：

```python
t = np.arange(14336) / 4096 - 0.5
```

因此分析区间从触发前 0.500 s 到触发后约 3.000 s。文件中的绝对事件时间不参与对齐，因为三台仪器的时钟相差 15–28 s，而各记录的峰值都出现在本机触发后约 0.023–0.055 s，表明设备绝对时钟没有可靠同步。

只处理原始振动速度，不微分为加速度，不执行默认滤波、重采样或波形峰值对齐。低频趋势由 STVMD 的零频残余项承载。

## 三次多通道分解

按以下顺序构造三个形状为 `(3, 14336)` 的输入数组：

```python
signals["Tran"] = np.vstack([data_5m["Tran"], data_10m["Tran"], data_15m["Tran"]])
signals["Vert"] = np.vstack([data_5m["Vert"], data_10m["Vert"], data_15m["Vert"]])
signals["Long"] = np.vstack([data_5m["Long"], data_10m["Long"], data_15m["Long"]])
```

每个数组的通道顺序固定为 5m、10m、15m。Notebook 依次运行 Tran、Vert、Long，结果存入以方向名为键的字典，避免变量互相覆盖。

## STVMD 算法与参数

使用仓库中的动态多通道 STVMD 定义：

- Hamming 窗；
- 输入边界使用反射填充；
- 相邻时窗步长固定为 1；
- 同一时窗内三个通道共享中心频率；
- 中心频率随时窗变化；
- ADMM 更新采用仓库中的归一化频率尺度；
- 频率中心均匀初始化；
- 收敛容差默认 `1e-9`；
- 拉格朗日更新参数 `tau` 默认 `1e-5`。

配置单元格提供以下用户参数：

```python
K = 4
ALPHA = 50.0
WINDOW_LENGTH = 64
MAX_ITERS = 2000
BATCH_WINDOWS = 256
PLOT_MAX_HZ = 200.0
SAVE_OUTPUTS = True
```

参数约束和说明：

- `K` 可调范围为 3–5，默认 4。
- 与仓库及论文图例一致，`K` 包含零频残余项，因此可绘制的非零中心频率模态数通常为 `K - 1`。
- `ALPHA` 默认 50，建议调节范围为 10–1000；只强制要求为有限正数。
- `WINDOW_LENGTH` 只允许仓库使用的候选值 `8、16、32、64、128、256`，默认 64。
- Notebook 明确显示当前时窗的时长和频率分辨率 `fs / WINDOW_LENGTH`，提醒用户较短时窗会降低频率分辨率。
- `MAX_ITERS`、`BATCH_WINDOWS` 和绘图最高频率可以修改，但不会改变步长为 1 的要求。

## 内存受控的分批实现

仓库原始 Notebook 会同时保存全部时窗的复数频谱和 ADMM 中间量。对 14336 点、三个通道和多个模态直接运行会产生较高内存占用。

新 Notebook 保留原更新方程，但沿时间窗维度分批：

1. 反射填充完整输入；
2. 按步长 1 生成当前批次的 Hamming 窗片段；
3. 对批次执行动态 STVMD 更新；
4. 将批次的模态时域片段累加到全局 overlap-add 缓冲区；
5. 保存该批次的动态中心频率；
6. 释放批次频谱和 ADMM 中间数组后处理下一批。

动态 STVMD 的不同时间窗中心频率彼此独立，同一批次内仍严格共享三个通道的信息。分批只限制同时驻留内存的时间窗数量；更新方程、窗函数、反射填充和步长均保持不变。各批次使用相同容差，结果应在该数值容差内与整批计算一致。

输出结构为：

```python
results[direction] = {
    "modes": ...,              # (K, 3, N)
    "center_freq_hz": ...,     # (K, N)
    "iterations": ...,         # 每批迭代次数
    "converged": ...,          # 每批收敛状态
    "reconstruction": ...,     # (3, N)
    "nrmse": ...,              # (3,)
    "energy_fraction": ...,    # (K, 3)
    "frequency_bands_hz": ...  # (K, 2)
}
```

## 平均时频谱

平均时频谱不对三个时域波形直接求平均。对每个通道使用与 STVMD 相同的 Hamming 窗和步长计算短时频谱，然后对功率求平均：

```python
power_mean = np.mean(np.abs(stft_channels) ** 2, axis=0)
power_db = 10 * np.log10(power_mean / power_mean.max() + eps)
```

该方式避免不同通道的相位抵消，并与多通道 STVMD 在中心频率更新中汇总通道能量的原则一致。三个通道不单独归一化，因此振幅较大的测点会贡献更多能量。

## 图形设计

Tran、Vert、Long 各生成相同的论文式图组。坐标单位使用速度 `mm/s`、时间 `s` 和频率 `Hz`。

### 图 1：输入速度与原始时频谱

- 左侧：5m、10m、15m 三通道速度波形；
- 右侧：三通道平均功率时频谱；
- 使用统一时间轴，标出触发时刻 `t = 0`。

### 图 2：STVMD 模态波形

- 行对应零频残余项和各非零中心频率模态；
- 列对应 5m、10m、15m；
- 所有子图共享时间轴；
- 标题明确区分 residual 和 mode。

### 图 3：动态中心频率与重构质量

- 绘制 `K - 1` 条非零动态中心频率轨迹；
- 对比原始信号与全部模态之和；
- 报告每个通道的归一化均方根误差；
- 报告各模态在三个通道中的能量占比；
- 不对中心频率轨迹做默认平滑，以保留算法原始输出。

### 图 4：频谱区域与瞬时频率对应关系

该图对应用户提供的2022论文参考图，并增加真实数据下的频带估计：

- 左侧窄面板绘制三个通道的合成功率频谱；
- 右侧宽面板绘制三通道平均功率时频谱；
- 在右侧叠加各非零模态的动态中心频率轨迹；
- 对每个模态，将三个通道重构模态的正频率功率相加；
- 从累计功率分布计算 5% 和 95% 分位频率，作为该模态的 90% 能量频带；
- 使用同一颜色把频带上下界同时画在左侧频谱和右侧时频谱中；
- 图例明确给出 Mode 编号和频带数值。

这样可以直接观察某条瞬时频率轨迹对应总体傅里叶频谱中的哪一段能量。

## 输出文件

Notebook 默认在 `output/stvmd_blast/` 下按方向保存：

- `tran_input_tf.png`、`vert_input_tf.png`、`long_input_tf.png`
- `*_modes.png`
- `*_if_reconstruction.png`
- `*_spectrum_if_mapping.png`
- `stvmd_results.npz`

PNG 使用 300 dpi。`NPZ` 保存配置、模态、中心频率、重构误差、能量占比和频带，便于后续复用。关闭 `SAVE_OUTPUTS` 时只在 Notebook 内显示结果。

## 校验、警告与失败处理

运行前检查：

- 三个输入文件存在；
- 能识别 Tran、Vert、Long 数据列；
- 样本均为有限数值；
- 三份数据采样率一致且为正；
- 每份数据至少包含一个完整时窗；
- K、α、时窗长度、批大小和最大迭代数满足约束。

如果输入无效，抛出包含文件名和原因的明确异常。如果某批次达到最大迭代数仍未收敛，保留计算结果并显示方向、批次、最终差值和迭代次数警告。绘图前检查结果形状及有限数值。

## 验证策略

实现采用测试优先：

1. 构造具有三个通道、两个共享时变频率分量的短合成信号；
2. 验证数据加载、共同长度截取和方向矩阵的通道顺序；
3. 验证分批结果的输出形状、有限数值和 overlap-add 重构；
4. 在小样本上比较单批与多批结果，要求差异落在数值容差内；
5. 验证对应模态在三个通道中共享中心频率轨迹；
6. 验证频带分位数单调且覆盖对应模态的主要谱峰；
7. 执行 Notebook 的轻量烟雾测试，确认所有单元格可按顺序运行并生成预期图形对象。

真实数据的完整三方向分解计算量较高，因此自动测试使用缩短的合成数据和真实数据片段；Notebook 保留运行完整14336点数据的主单元格。
